# Database Architecture (`db.md`)

This document defines the exhaustive database schema for the Farms Empire Web Platform, optimized for Turso (libSQL/SQLite) and Drizzle ORM.

## 1. Tables and Columns

### `users`

Stores internal staff accounts for the administrative dashboard.

| Column Name     | Type      | Nullable | Default       | Description                                  |
| :-------------- | :-------- | :------- | :------------ | :------------------------------------------- |
| `id`            | `text`    | NO       | `uuid()`      | Primary key. Unique identifier for the user. |
| `name`          | `text`    | NO       | -             | Full name of the staff member.               |
| `email`         | `text`    | NO       | -             | Unique email address used for login.         |
| `password_hash` | `text`    | NO       | -             | Bcrypt/Argon2 hashed password.               |
| `role`          | `text`    | NO       | `'MANAGER'`   | User role. Must be `ADMIN` or `MANAGER`.     |
| `created_at`    | `integer` | NO       | `unixepoch()` | Timestamp of account creation.               |
| `updated_at`    | `integer` | NO       | `unixepoch()` | Timestamp of last profile update.            |

### `blogs`

Stores dynamic blog posts for the public marketing site.

| Column Name       | Type      | Nullable | Default       | Description                                               |
| :---------------- | :-------- | :------- | :------------ | :-------------------------------------------------------- |
| `id`              | `text`    | NO       | `uuid()`      | Primary key.                                              |
| `title`           | `text`    | NO       | -             | The title of the blog post.                               |
| `slug`            | `text`    | NO       | -             | URL-friendly unique identifier (e.g., `farm-is-good-1`).  |
| `excerpt`         | `text`    | YES      | `NULL`        | Short summary for list views and SEO meta descriptions.   |
| `content`         | `text`    | NO       | -             | Full HTML content generated by the TipTap WYSIWYG editor. |
| `cover_image_url` | `text`    | YES      | `NULL`        | Public Cloudinary URL for the featured image.             |
| `status`          | `text`    | NO       | `'DRAFT'`     | Publication status. Must be `DRAFT` or `PUBLISHED`.       |
| `author_id`       | `text`    | NO       | -             | Foreign key referencing `users.id`.                       |
| `created_at`      | `integer` | NO       | `unixepoch()` | Timestamp of creation.                                    |
| `updated_at`      | `integer` | NO       | `unixepoch()` | Timestamp of last modification.                           |

## 2. Primary Keys and Foreign Key Relationships

### Primary Keys

Every table uses a `text` column named `id` as the Primary Key, populated with UUIDs at the application layer via Drizzle ORM.

### Foreign Keys

| Source Table | Source Column | Target Table | Target Column | On Delete Action | Description                                                                                                                 |
| :----------- | :------------ | :----------- | :------------ | :--------------- | :-------------------------------------------------------------------------------------------------------------------------- |
| `blogs`      | `author_id`   | `users`      | `id`          | `RESTRICT`       | Links a blog post to its author. `RESTRICT` prevents accidental deletion of a user without reassigning their content first. |
| `projects`   | `author_id`   | `users`      | `id`          | `RESTRICT`       | Links a project to its author. `RESTRICT` prevents accidental deletion of a user without reassigning their content first.   |

_Note: `careers` does not have an `author_id` foreign key as job postings are organizational, not individually authored in this context._

## 3. Indexes

Indexes are strategically placed to optimize Drizzle ORM queries, prevent N+1 issues, and respect Turso's read/write limits.

| Table           | Index Name                    | Column(s)              | Type     | Reason / Usage Context                                                         |
| :-------------- | :---------------------------- | :--------------------- | :------- | :----------------------------------------------------------------------------- |
| `users`         | `idx_users_email`             | `email`                | `UNIQUE` | Ensures no duplicate accounts; speeds up login queries.                        |
| `blogs`         | `idx_blogs_slug`              | `slug`                 | `UNIQUE` | Ensures unique URLs; speeds up public dynamic routing (`/blog/[slug]`).        |
| `blogs`         | `idx_blogs_status_created`    | `status`, `created_at` | `INDEX`  | Optimizes public list views (fetching only `PUBLISHED` blogs ordered by date). |
| `blogs`         | `idx_blogs_author`            | `author_id`            | `INDEX`  | Optimizes dashboard queries filtering blogs by a specific author.              |
| `projects`      | `idx_projects_slug`           | `slug`                 | `UNIQUE` | Ensures unique URLs; speeds up public dynamic routing.                         |
| `projects`      | `idx_projects_status_created` | `status`, `created_at` | `INDEX`  | Optimizes public list views for published projects.                            |
| `projects`      | `idx_projects_author`         | `author_id`            | `INDEX`  | Optimizes dashboard queries filtering projects by a specific author.           |
| `careers`       | `idx_careers_slug`            | `slug`                 | `UNIQUE` | Ensures unique URLs; speeds up public dynamic routing.                         |
| `careers`       | `idx_careers_status_created`  | `status`, `created_at` | `INDEX`  | Optimizes public list views for open careers.                                  |
| `contacts`      | `idx_contacts_is_read`        | `is_read`              | `INDEX`  | Optimizes dashboard inbox filtering (e.g., fetching only `unread` contacts).   |
| `contacts`      | `idx_contacts_created`        | `created_at`           | `INDEX`  | Optimizes chronological sorting in the dashboard inbox.                        |
| `subscriptions` | `idx_subscriptions_email`     | `email`                | `UNIQUE` | Prevents duplicate newsletter signups.                                         |
| `publications`  | `idx_publications_category`   | `category`             | `INDEX`  | Optimizes filtering publications by type on the public page.                   |

## 5. Row-Level Security (RLS) Rules

**RLS is NOT APPLICABLE for this architecture.**

While Turso/libSQL supports Row-Level Security, the Farms Empire platform enforces **Strict Server-Side RBAC** at the application layer using TanStack Start Server Functions and Better Auth sessions.

- All dashboard routes and data mutations are wrapped in server-side middleware that validates the user's session and role (`ADMIN` vs `MANAGER`) _before_ any Drizzle ORM query is executed.
- Client-side UI restrictions (hiding buttons) are purely for UX; the server functions act as the absolute security boundary.
- Implementing RLS in SQLite/libSQL for this specific stack would add unnecessary complexity and potential performance overhead without providing additional security beyond what the Server Functions already guarantee.

## 6. Seed Data Requirements

The database must be seeded upon the very first deployment to ensure the system is usable. This is handled by a dedicated seed script executed via a TanStack Start server function or CLI command.

### Mandatory Seed Data:

1. **Initial Super Admin User**:
   - `email`: Sourced from the `INITIAL_ADMIN_EMAIL` environment variable.
   - `password_hash`: Hashed value of the `INITIAL_ADMIN_PASSWORD` environment variable.
   - `role`: `ADMIN`
   - `name`: "System Administrator"
   - _Constraint_: The seed script must check if the `users` table is empty before executing to prevent duplicate admin creation on subsequent deployments.

### Optional/Recommended Seed Data (Based on Project Brief):

2. **Publications** (Pre-populating the Publications page):
   - "Communities as Agricultural Enterprises" (Category: `RESEARCH`)
   - "Idle Land, Active Returns" (Category: `POLICY_BRIEF`)
   - "Women at the Centre" (Category: `IMPACT_REPORT`)
   - "Africa's $350 Billion Opportunity" (Category: `THOUGHT_LEADERSHIP`)
   - "From Field to Table" (Category: `PRACTITIONER_GUIDE`)

3. **Projects** (Pre-populating the Projects page):
   - "The Pioneer Community Farm Initiative" (Location: Ogun State)
   - "Women's Agricultural Cooperative Development Programme" (Location: Oyo State)
   - "Absentee Landowner Partnership Programme" (Location: Multiple States)
   - "School Farm & Youth Agripreneur Initiative" (Location: Lagos State)

_Note: Environment variables containing the initial admin credentials must be securely deleted from the Cloudflare dashboard immediately after the first successful deployment and seeding._
